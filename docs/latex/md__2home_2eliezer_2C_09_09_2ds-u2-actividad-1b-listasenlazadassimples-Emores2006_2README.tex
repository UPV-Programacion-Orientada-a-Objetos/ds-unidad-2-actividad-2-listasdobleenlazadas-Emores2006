Una empresa de monitoreo de Infraestructura Crítica (IC) necesita un sistema de bajo nivel para registrar, almacenar y procesar lecturas de múltiples tipos de sensores (temperatura, presión, vibración) de manera unificada.

El sistema debe superar dos grandes limitaciones de diseño\+:


\begin{DoxyItemize}
\item {\bfseries{Rigidez del Tipo de Dato\+:}} Las lecturas pueden ser enteras (ej. conteo de vibraciones) o de punto flotante (ej. temperatura), por lo que el contenedor debe ser genérico.
\item {\bfseries{Rigidez de la Estructura\+:}} Se necesita una estructura de datos que se adapte al número variable de lecturas.
\end{DoxyItemize}

El objetivo es crear una Jerarquía de Clases Polimórfica que gestione Listas Enlazadas Simples Genéricas ({\ttfamily \doxylink{classListaSensor}{Lista\+Sensor}\texorpdfstring{$<$}{<}T\texorpdfstring{$>$}{>}}). El diseño debe permitir que cualquier tipo de sensor (subclase) pueda ser agregado a una única lista de gestión, forzando la implementación de métodos esenciales a través de Clases Abstractas.

\begin{DoxyItemize}
\item \texorpdfstring{$\ast$}{*}\texorpdfstring{$\ast$}{*}\+Clase Base Abstracta ({\ttfamily \doxylink{classSensorBase}{Sensor\+Base}})\+:\texorpdfstring{$\ast$}{*}\texorpdfstring{$\ast$}{*}
\begin{DoxyItemize}
\item Propósito\+: Define la interfaz común para todos los sensores.
\item Método Virtual Puro\+: {\ttfamily virtual void procesar\+Lectura() = 0;} y {\ttfamily virtual void imprimir\+Info() const = 0;}.
\item Atributo\+: {\ttfamily protected char nombre\mbox{[}50\mbox{]};} (Identificador del sensor).
\end{DoxyItemize}
\item {\bfseries{Clases Derivadas (Concretas)\+:}}
\begin{DoxyItemize}
\item {\ttfamily \doxylink{classSensorTemperatura}{Sensor\+Temperatura} \+: public \doxylink{classSensorBase}{Sensor\+Base}}\+: Representa un sensor que maneja lecturas de tipo {\ttfamily float}.
\item {\ttfamily \doxylink{classSensorPresion}{Sensor\+Presion} \+: public \doxylink{classSensorBase}{Sensor\+Base}}\+: Representa un sensor que maneja lecturas de tipo {\ttfamily int}.
\end{DoxyItemize}
\item {\bfseries{Clase de Nodo y Lista Genérica\+:}}
\begin{DoxyItemize}
\item {\ttfamily template \texorpdfstring{$<$}{<}typename T\texorpdfstring{$>$}{>} struct Nodo}\+: Estructura o clase que contiene un dato {\ttfamily T} y un puntero {\ttfamily Nodo\texorpdfstring{$<$}{<}T\texorpdfstring{$>$}{>}\texorpdfstring{$\ast$}{*} siguiente}.
\item {\ttfamily template \texorpdfstring{$<$}{<}typename T\texorpdfstring{$>$}{>} class \doxylink{classListaSensor}{Lista\+Sensor}}\+: Implementa las funciones de inserción, búsqueda y liberación de la Lista Enlazada Simple para el tipo de dato {\ttfamily T}.
\end{DoxyItemize}
\end{DoxyItemize}\hypertarget{md__2home_2eliezer_2C_09_09_2ds-u2-actividad-1b-listasenlazadassimples-Emores2006_2README_autotoc_md7}{}\doxysubsubsection{\texorpdfstring{Proceso de Solución Detallado}{Proceso de Solución Detallado}}\label{md__2home_2eliezer_2C_09_09_2ds-u2-actividad-1b-listasenlazadassimples-Emores2006_2README_autotoc_md7}

\begin{DoxyEnumerate}
\item {\bfseries{Lista de Gestión Polimórfica\+:}} El programa principal debe utilizar una Lista Enlazada Simple {\bfseries{NO}} Genérica (ej., {\ttfamily Lista\+General}) que almacene punteros a la clase base ({\ttfamily Sensor\+Base\texorpdfstring{$\ast$}{*}}). Esto permite guardar {\ttfamily Sensor\+Temperatura\texorpdfstring{$\ast$}{*}} y {\ttfamily Sensor\+Presion\texorpdfstring{$\ast$}{*}} juntos.
\item {\bfseries{Integración de la Lista Genérica\+:}} Cada clase concreta ({\ttfamily \doxylink{classSensorTemperatura}{Sensor\+Temperatura}}, {\ttfamily \doxylink{classSensorPresion}{Sensor\+Presion}}) debe contener como atributo una instancia de su respectiva Lista Genérica.
\begin{DoxyItemize}
\item {\ttfamily \doxylink{classSensorTemperatura}{Sensor\+Temperatura}} contendrá {\ttfamily \doxylink{classListaSensor}{Lista\+Sensor}\texorpdfstring{$<$}{<}float\texorpdfstring{$>$}{>} historial;}
\item {\ttfamily \doxylink{classSensorPresion}{Sensor\+Presion}} contendrá {\ttfamily \doxylink{classListaSensor}{Lista\+Sensor}\texorpdfstring{$<$}{<}int\texorpdfstring{$>$}{>} historial;}
\end{DoxyItemize}
\item {\bfseries{Registro de Lecturas\+:}} Implementar un método para cada clase derivada que permita añadir una nueva lectura a su lista interna.
\item {\bfseries{Procesamiento Polimórfico\+:}} La función principal del sistema debe iterar sobre la Lista de Gestión Polimórfica (que almacena {\ttfamily Sensor\+Base\texorpdfstring{$\ast$}{*}}) y llamar al método {\ttfamily procesar\+Lectura()}.
\begin{DoxyItemize}
\item Para {\ttfamily \doxylink{classSensorTemperatura}{Sensor\+Temperatura}}, el método {\ttfamily procesar\+Lectura()} podría calcular y eliminar el valor más bajo de su lista interna.
\item Para {\ttfamily \doxylink{classSensorPresion}{Sensor\+Presion}}, podría calcular el promedio de las lecturas.
\end{DoxyItemize}
\item {\bfseries{Liberación de Memoria\+:}} El destructor del sistema debe\+:
\begin{DoxyItemize}
\item Liberar los nodos de la Lista de Gestión Polimórfica (la lista principal).
\item Para cada {\ttfamily Sensor\+Base\texorpdfstring{$\ast$}{*}} liberado, debe invocar al destructor de la clase derivada.
\item Los destructores de las clases derivadas ({\ttfamily \doxylink{classSensorTemperatura}{Sensor\+Temperatura}}, {\ttfamily \doxylink{classSensorPresion}{Sensor\+Presion}}) son responsables de liberar la memoria de todos los nodos de su {\ttfamily \doxylink{classListaSensor}{Lista\+Sensor}\texorpdfstring{$<$}{<}T\texorpdfstring{$>$}{>}} interna.
\end{DoxyItemize}
\end{DoxyEnumerate}

Para llevar a cabo esta actividad, se deberá investigar y comprender a fondo\+:


\begin{DoxyEnumerate}
\item Regla de los Tres/\+Cinco con Templates\+: La implementación obligatoria del Destructor, Constructor de Copia y Operador de Asignación en clases genéricas que gestionan memoria dinámica (como {\ttfamily \doxylink{classListaSensor}{Lista\+Sensor}\texorpdfstring{$<$}{<}T\texorpdfstring{$>$}{>}}) para evitar fugas y copias superficiales.
\item Polimorfismo con Destructores Virtuales\+: La necesidad crítica de declarar el destructor de la clase base (\texorpdfstring{$\sim$}{\string~}\+Sensor\+Base()) como virtual para garantizar que se llame al destructor correcto de la clase derivada al liberar un puntero de la base, evitando fugas de memoria en las clases derivadas.
\item Técnicas de Downcasting Seguro\+: Si bien no es estrictamente necesario, investigar el uso de dynamic\+\_\+cast para acceder a propiedades específicas de la subclase si fuera requerido fuera del polimorfismo básico (solo para enriquecer la investigación).
\item Sintaxis de Clases y Structs Anidados\+: Cómo definir el struct Nodo dentro de la \doxylink{classListaSensor}{Lista\+Sensor} para mantener el encapsulamiento.
\item Instalación, configuración y uso de CMake para generar sistemas redistribuibles.
\end{DoxyEnumerate}
